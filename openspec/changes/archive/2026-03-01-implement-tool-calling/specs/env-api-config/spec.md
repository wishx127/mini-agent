## 新增需求

### 需求：配置支持工具特定设置
ModelConfig 必须包含 `tools` 配置对象来管理工具特定设置。

#### 场景：获取工具配置
- **当** 调用方执行 `configManager.getConfig().tools`
- **则** 返回包含工具特定设置的 ToolsConfig 对象

### 需求：环境变量启用/禁用工具
配置系统必须支持 `TOOL_<NAME>_ENABLED` 环境变量来启用/禁用工具。

#### 场景：通过环境变量启用工具
- **当** 设置环境变量 `TOOL_TAVILY_ENABLED=true`
- **则** Tavily 工具注册为启用状态

#### 场景：通过环境变量禁用工具
- **当** 设置环境变量 `TOOL_TAVILY_ENABLED=false`
- **则** Tavily 工具注册为禁用状态

### 需求：配置从环境变量加载工具 API 密钥
配置系统必须从环境变量加载工具特定的 API 密钥。

#### 场景：Tavily API 密钥已配置
- **当** 设置环境变量 `TAVILY_API_KEY`
- **则** API 密钥在工具配置中可用

### 需求：默认工具配置
当未提供工具特定配置时，配置系统必须提供合理的默认值。

#### 场景：未提供工具配置
- **当** 未设置任何工具相关的环境变量
- **则** 所有工具默认为禁用状态

### 需求：配置验证所需的工具凭证
配置系统必须验证工具启用时所需的凭证是否已设置。

#### 场景：启用工具但未设置 API 密钥
- **当** 通过环境变量启用工具但未设置所需的 API 密钥
- **则** 配置验证失败，提示缺少 API 密钥的错误信息

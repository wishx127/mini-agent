## 新增需求

### 需求：Agent 检测何时需要使用工具
Agent 必须分析用户输入以确定是否需要外部工具协助。

#### 场景：LLM 返回工具调用
- **当** 用户输入需要外部信息且 LLM 支持函数调用
- **则** Agent 从 LLM 接收工具调用并执行相应工具

#### 场景：LLM 未返回工具调用
- **当** 用户输入可能需要工具但 LLM 未返回工具调用
- **则** Agent 使用基于规则的兜底机制判断是否应调用工具

### 需求：Agent 使用基于规则的兜底机制处理常见模式
当 LLM 未指示使用工具时，Agent 必须使用预定义的关键词匹配作为兜底。

#### 场景：查询包含搜索关键词
- **当** 用户输入包含 "搜索"、"查询"、"联网"、"最新"、"news" 等关键词
- **则** Agent 自动调用搜索工具

### 需求：Agent 执行工具并将结果包含在上下文中
Agent 必须执行选定的工具并将结果包含在 LLM 上下文中以生成最终响应。

#### 场景：工具执行成功
- **当** 工具执行成功
- **则** Agent 将工具结果包含在对话上下文中并生成响应

#### 场景：工具执行失败
- **当** 工具执行抛出错误
- **则** Agent 捕获错误，将错误信息包含在上下文中，并生成承认失败的响应

### 需求：Agent 限制工具调用迭代次数
Agent 必须通过设置最大迭代次数来防止无限工具调用循环。

#### 场景：达到最大迭代次数
- **当** 工具在单次对话中被调用超过 3 次
- **则** Agent 停止调用工具并返回累积的结果

### 需求：Agent 在调用前检查工具可用性
Agent 必须在尝试使用工具前验证所需工具已启用并配置。

#### 场景：所需工具已禁用
- **当** 用户请求搜索但 Tavily 工具通过配置禁用
- **则** Agent 在不使用工具的情况下继续执行并通知用户工具已禁用

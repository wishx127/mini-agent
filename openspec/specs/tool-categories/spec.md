# 工具分类规范

## 新增需求

### 需求：预定义分类类型

系统必须支持五种预定义的工具分类。

#### 场景：INTERNAL 分类（内部工具）

- **当** 工具被分类为 INTERNAL 时
- **则** 它应被标记为内部操作，如数据库查询
- **且** 应可被内部系统使用

#### 场景：EXTERNAL_API 分类（外部 API）

- **当** 工具被分类为 EXTERNAL_API 时
- **则** 它应被标记为外部 API 调用，如网络搜索
- **且** 应考虑速率限制

#### 场景：FILE_SYSTEM 分类（文件系统）

- **当** 工具被分类为 FILE_SYSTEM 时
- **则** 它应被标记为文件读写操作
- **且** 应有适当的安全考虑

#### 场景：VECTOR_SEARCH 分类（向量检索）

- **当** 工具被分类为 VECTOR_SEARCH 时
- **则** 它应被标记为语义搜索和检索操作
- **且** 应针对相似性搜索查询进行优化

#### 场景：SANDBOX 分类（代码执行沙箱）

- **当** 工具被分类为 SANDBOX 时
- **则** 它应被标记为沙箱环境中的代码执行
- **且** 应有适当的超时和资源限制

### 需求：工具分类赋值

工具必须支持通过配置进行分类赋值。

#### 场景：单分类赋值

- **当** 工具的 category 设置为单个值时
- **则** 工具应注册在该分类下

#### 场景：多分类赋值

- **当** 工具的 categories 设置为多个值时
- **则** 工具应注册在所有指定的分类下
- **且** 应可从每个分类中查询

#### 场景：未指定分类

- **当** 工具没有指定分类时
- **则** 工具应被分类为 UNCATEGORIZED
- **且** 仍可通过普通注册表查询访问

### 需求：按分类查询

注册表必须支持按分类查询工具。

#### 场景：按单个分类查询工具

- **当** 调用 getToolsByCategory 并传入一个分类时
- **则** 应返回属于该分类的所有工具

#### 场景：按多个分类查询工具

- **当** 调用 getToolsByCategory 并传入多个分类时
- **则** 应返回属于任一指定分类的工具

### 需求：权限控制预留（未来扩展）

分类类型必须预留角色权限字段，供将来扩展使用。

#### 场景：分类定义包含角色字段

- **当** 分类类型定义时
- **则** 应包含可选的 `roles` 字段
- **且** 该字段用于指定访问该分类所需的角色列表

#### 场景：预留权限检查接口

- **当** 将来需要实现权限控制时
- **则** 应可通过 `hasAccess(category, userRoles)` 方法进行权限验证
- **且** 当前实现应返回 true（不做实际检查）
